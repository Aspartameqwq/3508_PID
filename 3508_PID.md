## 1. 项目概述

本项目实现 **STM32F407** 对 **DJI M3508 直流无刷减速电机**（搭配 C620 电调）的闭环伺服控制，支持 **速度环** 和 **位置环** 双模式，并通过 **DJI DR16 遥控器** 进行无线操控。

### 关键参数

| 项目 | 数值 |
|------|------|
| MCU 主频 | 168 MHz（HSI → PLL） |
| 控制周期 | **10 ms**（`CONTROL_PERIOD_MS`） |
| 电机数量 | **4**（CAN ID 1~4, `MOTOR_COUNT`） |
| 电机型号 | M3508 (C620 电调) |
| 编码器分辨率 | 8192 CPR（0~8191） |
| CAN 波特率 | 1 Mbps |
| DBUS 波特率 | 100 Kbps, 9-bit (8 data + even parity) |
| 遥控频率 | ~140 Hz（约 7ms 帧间隔） |
| 底盘构型 | X型全向轮（4轮切线布局） |
| 编译器 | ARM GCC (arm-none-eabi-gcc) |
| 构建系统 | CMake + CubeMX |

### 控制模式

**电机控制模式**（`control.h`，用于 `MotorControl_SetControlMode()`）：

| 模式 | 宏 | 值 | 说明 |
|------|-----|-----|------|
| **速度环** | `MOTOR_CONTROL_MODE_SPEED` | 0 | 仅速度 PID，目标速度 rad/s |
| **位置环** | `MOTOR_CONTROL_MODE_POSITION` | 1 | 位置 PID 外环 → 速度 PID 内环（级联），目标角度 ° |

**快捷调试模式**（`control.h`，用于 `debug_quick_mode` 变量）：

| 模式 | 宏 | 值 | 说明 |
|------|-----|-----|------|
| **速度环** | `DEBUG_QUICK_MODE_SPEED` | 0 | 用 `debug_set_speed[i]` 作目标 |
| **位置环** | `DEBUG_QUICK_MODE_POSITION` | 1 | 用 `debug_set_position[i]` 作目标 |
| **遥控器** | `DEBUG_QUICK_MODE_REMOTE` | 2 | 全向轮运动学解算 + 制动（默认） |

> **注意**: 遥控器模式下，左摇杆（ch2左右、ch3上下）控制底盘平移，右摇杆（ch0左右）控制旋转。经过运动学解算后每电机独立目标，支持底盘级两段式制动。

## 4. 模块详解

### 4.1 PID 控制器 (`pid.c`)

**文件**: [Core/Inc/pid.h](Core/Inc/pid.h), [Core/Src/pid.c](Core/Src/pid.c)

位置无关的通用 PID 控制器，支持：

- **积分分离**: 误差超出 `integral_separation_threshold` 时暂停积分，避免大幅扰动引起积分累积
- **输出限幅**: 输出钳位至 `[-output_limit, +output_limit]`
- **积分限幅**: 积分项独立钳位至 `[-integral_limit, +integral_limit]`
- **反计算抗饱和 (Back-Calculation Anti-Windup)**: 输出被限幅时，反算积分值使其恰好达到限幅边界

```c
typedef struct {
    float kp, ki, kd;                          // PID 增益
    float integral;                             // 积分累加项（内部状态）
    float prev_error;                           // 上次误差（内部状态）
    float output_limit;                         // 输出幅值限制
    float integral_limit;                       // 积分幅值限制
    float integral_separation_threshold;        // 积分分离阈值
} PIDController;
```

**API**:

| 函数 | 功能 |
|------|------|
| `PID_Init(pid, kp, ki, kd, out_lim, int_lim, sep)` | 初始化参数与清零状态 |
| `PID_Calculate(pid, setpoint, measurement, dt_s)` | 计算一次 PID 输出 |
| `PID_Reset(pid)` | 清零积分与历史误差 |
| `PID_Clamp(value, limit)` | 对称限幅工具函数 |

### 4.2 电机控制层 (`control.c`)

**文件**: [Core/Inc/control.h](Core/Inc/control.h), [Core/Src/control.c](Core/Src/control.c)

级联 PID 电机控制器，核心逻辑：

#### 控制律

每电机（i=0~3, CAN ID=i+1）独立执行：

```
位置环 (可选, P-only 常用):
  position_error = ShortestArc(target_deg[i], actual_deg[i])   // [−180°, +180°]
  speed_sp[i] = PID(position_kp[i], position_error, 0, dt_s)

速度环 (PI + 前馈 + 过零制动):
  torque_iq[i] = PID(speed_kp[i], speed_sp[i], actual_speed[i], dt_s)
               + ff_gain[i] × speed_sp[i]                       // 前馈补偿
               ± zero_cross_brake[i]                            // 过零反向制动

电流发送 (4路打包一帧):
  iq_clamped[i] = clamp(torque_iq[i], ±10000)
  CAN_Send(0x200, iq[0], iq[1], iq[2], iq[3])                 // 一次发送 4 路
```

#### 启动探测

无反馈时自动注入探测电流 `MOTOR_STARTUP_CURRENT = 2500`（iq）至对应电机，4 电机同时探测。每电机独立计时，超时 1500ms 后该电机停止探测并保持 iq=0。各电机 `feedback_valid[i]` 在收到首帧 CAN 反馈后置 1。

#### 过零处理

- **过零静止区** (`zero_cross_threshold`): 目标速度与实际速度同时低于阈值时，清零积分并输出 0，抑制零速抖动
- **过零制动** (`zero_cross_brake`): 目标与实际方向相反时追加制动电流，帮助快速换向

#### API

| 函数 | 功能 |
|------|------|
| `Control_Init()` | 初始化 4×双 PID + 反馈状态 |
| `Control_Task(dt_s)` | 主控制周期（遍历4电机：位置环→速度环→CAN发送） |
| `Control_OnCanRxMessage()` | CAN 反馈帧解析→路由到对应电机槽位[0~3] |
| `MotorControl_SetTargetAngle(id, deg)` | 设置指定电机位置环目标角度 (0~3) |
| `MotorControl_SetTargetSpeed(id, rad/s)` | 设置指定电机速度环目标速度 |
| `MotorControl_SetControlMode(id, mode)` | 切换指定电机速度环/位置环 |
| `MotorControl_ResetAngle(id)` | 指定电机角度清零（以当前位置为 0°） |
| `Control_StartTurns(id, turns)` | 指定电机启动转圈命令 |
| `Control_ResetPidState()` | 清零全部 8 个 PID 实例积分与历史误差 |

### 4.3 DR16 遥控器接收 (`DR16.c`)

**文件**: [Core/Inc/DR16.h](Core/Inc/DR16.h), [Core/Src/DR16.c](Core/Src/DR16.c)

#### 工作原理

```
DR16接收机 ──USART3(PC11)──► DMA1_Stream1 双缓冲循环模式
  sbus_rx_buffer[0][18]  ←→  sbus_rx_buffer[1][18]
         ↑ CT=0                    ↑ CT=1
         
  每完成 18 字节传输，CT 自动翻转 → IDLE 中断触发
  → 处理 CT 指向的缓冲（刚填满的那个）
  → 重置 DMA 目标为另一缓冲
```

#### 关键细节

- **为何用 IDLE 中断而非 DMA TC 中断**: DBUS 使用帧间空闲（>=1 字节时间的高电平）标识帧边界；USART IDLE 在 RX 线保持高电平 1 字节时间后触发，天然匹配此协议。
- **为何直接操作寄存器而非 HAL**: CubeMX 生成的是普通 DMA 模式，而 DBUS 需要双缓冲循环模式（`CIRC + DBM`），`HAL_DMAEx_MultiBufferStart()` 与 UART IDLE 配合时有竞态问题。
- **DR16_Init() 必须做的事情**: CubeMX 初始化遗留了一个坑——DMA 外设地址（PAR）不设，需手动写入 `dma_stream->PAR = (uint32_t)&USART3->DR`。

#### 数据结构

```c
RC_CtrlData.rc.ch0~ch3    // 摇杆通道 364~1684 (中位 ~1024)
RC_CtrlData.rc.s1, s2     // 开关: 1=UP, 3=MID, 2=DOWN
RC_CtrlData.mouse.x,y,z   // 鼠标速度
RC_CtrlData.key.v         // 键盘按键位掩码
```

#### DBUS 诊断流程（Ozone 中观察）

```
debug_dr16_idle_count  ↑     ← IDLE 中断正常触发
debug_dr16_frame_count ↑     ← RemoteDataProcess 有在跑
debug_dr16_raw[18]           ← 最近一帧原始 18 字节（检查协议对否）
debug_dr16_ct                ← 0 或 1，DMA 双缓冲目标指针
debug_dr16_ndtr              ← DMA 剩余计数（满帧 = 0）
debug_dr16_uart_sr           ← USART3 SR 寄存器（PE/FE/ORE/NE 错误位）
```

### 4.4 底盘运动学 (`chassis_kinematics.c`)

**文件**: [Core/Inc/chassis_kinematics.h](Core/Inc/chassis_kinematics.h), [Core/Src/chassis_kinematics.c](Core/Src/chassis_kinematics.c)

X型全向轮底盘运动学解算模块，负责遥控器通道→底盘速度→电机转速的完整映射链。

#### 轮组布局（俯视图）

```
  电机 3 (ID 4)              电机 0 (ID 1)
  左前 (-L,+L)               右前 (+L,+L)
  滚动 225°                  滚动 135°
  n=[-1/√2, -1/√2]          n=[-1/√2, +1/√2]

  电机 2 (ID 3)              电机 1 (ID 2)
  左后 (-L,-L)               右后 (+L,-L)
  滚动 315°                  滚动 45°
  n=[+1/√2, -1/√2]          n=[+1/√2, +1/√2]
```

#### API

| 函数 | 功能 |
|------|------|
| `RemoteToChassis(ch0~ch3, &vx, &vy, &ω)` | 遥控器通道→底盘速度（纯映射含死区） |
| `ChassisToMotors(vx, vy, ω, motor[4])` | 逆运动学：底盘速度→4电机目标转速 |
| `MotorsToChassis(motor[4], &vx, &vy, &ω)` | 前向运动学：4电机实际转速→底盘速度 |

#### 逆运动学公式

```
K = gear_ratio / (√2 × wheel_radius)

motor[0] = K·(-vx + vy + 2L·ω)    // 右前 ID1
motor[1] = K·(+vx + vy + 2L·ω)    // 右后 ID2
motor[2] = K·(+vx - vy + 2L·ω)    // 左后 ID3
motor[3] = K·(-vx - vy + 2L·ω)    // 左前 ID4
```

#### 硬件补偿（在 main.c 遥控器路径中执行，不污染运动学库函数）

| 补偿项 | 变量 | 默认值 | 说明 |
|--------|------|--------|------|
| 偏向角 | `debug_kinematics_bias_angle_deg` | 0° | 旋转 (vx,vy) 修正机械偏斜 |
| CCW 偏航 | `debug_kinematics_ccw_correction_gain` | 0.4 | ω += gain × \|v\| |
| CCW 对角死区 | `debug_kinematics_ccw_dead_angle_deg` | 15° | 速度方向接近对角线时跳过 CCW |

### 4.5 调试与诊断 (`debug.c`)

**文件**: [Core/Inc/debug.h](Core/Inc/debug.h), [Core/Src/debug.c](Core/Src/debug.c)

所有调试变量均为 **`volatile` 全局变量**，可在 Ozone watch 窗口中直接观察甚至在线修改。下一控制周期（≤10ms）自动生效。

#### 模式切换入口

**`debug_quick_mode`** 选择当前调试模式：

```
debug_quick_mode = 0  →  速度环 (debug_set_speed)
debug_quick_mode = 1  →  位置环 (debug_set_position)  ← 默认
debug_quick_mode = 2  →  遥控器速度 (ch3 摇杆)
debug_quick_mode = 3  →  遥控器位置 (ch2 摇杆)
```

#### 转圈命令（独立触发，不受 quick_mode 影响）

**`debug_run_turns`** 设非零值即刻执行，完成后自动归零。

```
Ozone 设 debug_run_turns = 2.0  →  电机输出轴转 2 圈 → 自动停止 → 变量归零
```

实现原理："移动胡萝卜"——每周期将目标角度设在当前角度前方 30°，位置环持续追赶；总行程用编码器原始脉冲累计（正确处理 0↔8191 绕回），不受角度 360° 取模影响。速度由 `debug_turns_speed_rad_s` 直接控制，绕过位置环输出限幅。

## 8. PID 参数整定指南

### 调参入口

所有 PID 参数均定义在 [Core/Src/debug.c](Core/Src/debug.c) 中，通过 [Core/Src/control.c](Core/Src/control.c) 的 `Control_ApplyPidParams()` 每个控制周期自动同步到运行中的 PID 实例。

**Ozone 在线调参流程**：
1. 烧录固件 → Ozone 连接 → 暂停 (Break)
2. Watch 窗口添加 `debug_speed_*` / `debug_position_*` 变量
3. 双击变量值在线修改 → Continue 运行
4. 观察 `debug_actual_*` 变量确认效果
5. 满意后将 Ozone 中的最终参数回写到 `debug.c`

### 8.1 速度环整定

```
推荐顺序: Kp → Kd (含滤波) → Ki → 静摩擦补偿 → 过零参数

① debug_speed_kp (比例): 从 50 开始，逐步增加到对阶跃响应有明显跟踪
② debug_speed_kd (微分): 从 0.1 开始，配合 debug_speed_derivative_alpha (0.10~0.25)
   抑制 Kp 带来的超调与振荡。alpha 越小滤波越强（默认 0.15, 截止 ~2.4 Hz@10ms）
③ debug_speed_ki (积分): 从 1.0 开始微调，消除稳态误差。过大引起低频振荡
④ debug_speed_ff_gain (前馈): 一般保持 0（已确认 FF=0 更稳定）
⑤ debug_speed_stiction_comp_iq (静摩擦补偿): 仅位置环启用。电机卡住时注入额外电流
   （当 |speed_sp|>0.5 且 |actual|<0.2 时触发，接近目标时线性缩减至 0）
⑥ debug_speed_zero_cross_threshold: 零速死区，减小静止抖动
⑦ debug_speed_zero_cross_brake: 提升方向切换响应，过大造成过冲
```

### 8.2 位置环整定

```
保守设计原则: 纯 P + 线性死区, Kd=0, Ki=0

① debug_position_kp: 从 0.5 开始，越大定位越快。当前 1.5（1° 误差→1.5 rad/s）
② debug_position_kd: 保持 0（级联结构中速度环已提供阻尼，再加 D 会放大噪声）
③ debug_position_ki: 保持 0（位置环无需消除静差，速度环已处理）
④ debug_position_deadband_deg: 线性死区 (当前 3°)，|error|≤死区→认为已到位
⑤ debug_position_output_limit: 限制位置环输出的最大速度指令 (当前 4.0 rad/s)
⑥ debug_position_stiction_threshold_deg: 静摩擦补偿缩减阈值 (当前 10°)，
   有效误差 < 此值时补偿量线性缩减至 0，避免微调时抖动

线性死区 vs 软着陆:
  线性: effective_error = |error| - deadband（连续过渡）
  软着陆: effective_error 在死区边界骤降→速度墙→阶梯爬行（已弃用）
```

### 8.3 整定常见问题

| 症状 | 可能原因 | 调整方向 |
|------|----------|----------|
| 电机高频啸叫/抖动 | Kd 过大或 D 项滤波不足 | ↓ `debug_speed_kd` 或 ↓ `debug_speed_derivative_alpha` |
| 间歇性高速振荡 | D 项噪声放大 | ↓ `debug_speed_derivative_alpha` (加强滤波) |
| 稳态误差无法消除 | Ki 过小 | ↑ `debug_speed_ki` |
| 低速/零速来回振荡 | Ki 过大或过零阈值不合适 | ↓ Ki 或 ↑ `zero_cross_threshold` |
| 定位过冲严重 | 位置环 Kp 过大 | ↓ `debug_position_kp` |
| 位置环边界振荡 (354°↔0°) | 过零震荡 | 自动处理 (ShortestArcError + 死区) |
| 电机卡住不动 (torque 400+ 不转) | 静摩擦 | ↑ `debug_speed_stiction_comp_iq` (默认 500) |
| 微调时抖动/冲击 | 静摩擦补偿过大 | ↑ `debug_position_stiction_threshold_deg` (更早退出) |
| 阶梯状爬行 (stick-slip) | 软着陆速度墙 | 确认使用线性死区 (当前已是) |
| 方向切换缓慢 | 过零制动太小 | ↑ `debug_speed_zero_cross_brake` |
| 转圈命令无效 | CAN 未就绪 | 检查 `debug_feedback_valid[i] = 1` |
| 电机不转（无反馈） | CAN 连接 / 电机 ID | 检查 `debug_can_rx_raw_count` 是否递增 |
| 推正前车往左/右偏 | 机械偏斜 | Ozone 调 `debug_kinematics_bias_angle_deg`（正值=CCW修正） |
| 直行时自动旋转 | CCW 修正过大 | ↓ `debug_kinematics_ccw_correction_gain` |
| 对角线推杆时过度旋转 | 对角死区太小 | ↑ `debug_kinematics_ccw_dead_angle_deg` |
| 摇杆回中后车滑太远 | 制动太弱 | ↑ `debug_brake_reverse_speed_ms` 或 ↑ `debug_brake_duration_per_ms` |
| 制动时车反向弹跳 | 制动太猛 | ↓ `debug_brake_reverse_speed_ms` 或 ↓ `debug_brake_duration_per_ms` |
| 静止后制动抖动 | 反向制动对噪声敏感 | 设 `debug_brake_reverse_speed_ms=0` 和 `debug_brake_reverse_omega_rad_s=0` |
| 轻推摇杆误触发制动 | 制动阈值太大 | ↓ `debug_brake_threshold_rad_s` |
| 摇杆回中制动迟迟不触发 | 制动阈值太小 | ↑ `debug_brake_threshold_rad_s` |

### 8.4 快捷调试模式切换指南

#### 模式总览

所有模式通过 **`debug_quick_mode`**（Ozone 直接修改）切换。

| `debug_quick_mode` | 宏 | 行为 | 目标来源 |
|---------------------|-----|------|----------|
| **0** | `DEBUG_QUICK_MODE_SPEED` | 速度环 | `debug_set_speed[i]` (rad/s) |
| **1** | `DEBUG_QUICK_MODE_POSITION` | 位置环 | `debug_set_position[i]` (°) |
| **2** | `DEBUG_QUICK_MODE_REMOTE` | 遥控器（默认） | 全向轮运动学解算 |

#### 各模式详解

**模式 0 — 速度环 (Speed)**

```
Ozone 设 debug_quick_mode = 0
     设 debug_set_speed[0] = 5.0   → 电机1 以 5.0 rad/s 旋转
     设 debug_set_speed[1] = 0.0   → 电机2 硬停
     设 debug_set_speed[2] = -3.0  → 电机3 反向旋转
（每个电机独立目标，4电机可同时不同速）
```

**模式 1 — 位置环 (Position)**

```
Ozone 设 debug_quick_mode = 1
     设 debug_set_position[0] = 90.0 → 电机1 转到 90° 并保持
     设 debug_set_position[1] = 0.0  → 电机2 转回 0°
```

**模式 2 — 遥控器 (Remote)**（默认）

```
Ozone 设 debug_quick_mode = 2
     操作遥控器:
       左摇杆 ch2（左右）→ X 平移    右摇杆 ch0（左右）→ 旋转
       左摇杆 ch3（上下）→ Y 平移    右摇杆 ch1（上下）→ 预留

     流程: 摇杆→底盘速度→硬件补偿→逆运动学→4电机→制动判断
     硬件补偿:
       - 偏向角: 旋转 (vx,vy) 修正方向偏移
       - CCW 偏航: ω += gain × |v|，对角线方向自动跳过
```

#### 独立触发 — 转圈命令

**不依赖 `debug_quick_mode`**，任何模式下均可使用：

```
Ozone 设 debug_run_turns = 2.0   → 输出轴从当前位置转 2 圈后停止
Ozone 设 debug_run_turns = -0.5  → 输出轴反向转半圈
```

执行后 `debug_run_turns` 自动归零。转速由 `debug_turns_speed_rad_s`（默认 40 rad/s 电机端）控制。减速比 `debug_gear_ratio`（默认 3591/187）决定电机→输出轴换算。

> **注意**: 转圈命令使用编码器脉冲累计判定完成，内部自动切到位置环。如果事后需要切换模式，直接改 `debug_quick_mode` 即可。

| DMA RX | **DMA1 Stream 1, Channel 4, Normal Mode** |

> **注意**: CubeMX 中 DMA 选择 **Normal** 模式即可。`DR16_Init()` 会在运行时将其升级为 Circular + Double-Buffer 模式。

### TIM6

| 参数 | 设置 |
|------|------|
| Prescaler | 83 (84 MHz → 1 MHz) |
| Period | 999 (1 kHz → 1ms) |
| Auto-reload | Enable |

### GPIO

| 引脚 | 功能 | 备注 |
|------|------|------|
| PD0 | CAN1_RX | — |
| PD1 | CAN1_TX | — |
| PC11 | USART3_RX | DBUS 输入 |

### NVIC

| 中断 | 优先级 |
|------|--------|
